Welcome to my guitar Auto Tabber!

Enter the notes below, in GUITAR FORMAT (see example input below).
Please note that the only reason why guitar format is required is because it is the most confortable format to test the app; the app does not 'cheat'.
Once you input the notes, click the "Submit" button to run the Auto Tabber.
The Auto Tabber will try to find a decent arrangement for the given score on guitar. Furthermore, it will try to split the notes into phrases for clarity.
The Auto Tabber uses a Probabilistic Graphical Model (more precisely, a Hidden Markov Model) to find the best arragement it can.
Please note, I only spent two days working on this project, it doesn't have much testing. It might do weird things! In particular, the Auto Tabber does not support bends, hammer-ons, pull-offs, harmonics or tapping. It only goes as far as the example below.
Check the code at: https://github.com/sprillo/web-app/blob/master/autoTabber.py
NOTE: When several notes are played at the same time, input them in order from lowest to highest string.



Try playing with these two model parameters. It might make the Auto Tabber produce better results.
New phrase penalty: (Default = 10)
Pinky penalty: (Default = 1)

New phrase penalty: This is how much Auto Tabber will penalize starting new phases. If you set this to 0 it will think every note is a phrase and there will be a lot of left hand movement. If you set this to 1000 it will try to fit all the piece into one phrase and there will be very little left hand movement.

Pinky penalty: This is how much the model penalizes using the pinky.
